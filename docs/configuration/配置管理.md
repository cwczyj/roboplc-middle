# 配置管理

> RoboPLC 支持配置热更新，无需重启系统即可应用配置更改。

## 🔧 热更新流程

### 触发方式

**配置文件变化** → ConfigLoader 检测 → 通过 Hub 广播 ConfigUpdate

### 更新流程

```
配置文件变化
  │
  ▼
  │
  ├── ConfigLoader 检测到 config.toml 变化
  │
  ├── 读取新配置
  │  │  │
  ├── 生成差异 JSON（新旧配置对比）
  │  │  │
  └── 通过 Hub 广播 ConfigUpdate 消息
  │
  │
  ▼
  │
  ├── ConfigUpdater 接收 ConfigUpdate
  │  │  │
  ├── 解析配置 JSON
  │  │  │
  ├── 更新全局 Config 变量
  │  │  ├── 更新 device_states 中的设备状态
  │  │  │  ├── 更新 DeviceManager 的 worker_map
  │  │  │
  └── 必要时通知相关 Worker 重启
  │
  └── 通过 Hub 发送 SystemStatus 确认更新
```

## 📋 ConfigUpdate 消息

**字段：**
```rust
pub struct ConfigUpdate {
    pub config: String,  // JSON 格式的配置内容
}
```

**用途：**
- 传递完整的新配置（包含 devices 配置）
- ConfigUpdater 可以直接对比新旧配置，决定需要更新的设备

## 🔧 配置更新逻辑

### 1. 配置解析
ConfigUpdater 使用 `serde_json::from_str()` 解析配置：
```rust
let new_config: Config = serde_json::from_str(&config_json)?;
```

### 2. 差异检测

遍历新旧配置的差异：
```rust
// 添加的设备
let added_devices = new_config.devices.iter()
    .filter(|d| !old_config.devices.iter().any(|od| od.id == d.id));

// 移除的设备
let removed_devices = old_config.devices.iter()
    .filter(|d| !new_config.devices.iter().any(|nd| nd.id == d.id));
```

### 3. 状态更新

#### 更新 device_states
```
// 移除的设备：从 device_states 中删除并记录断开
for device_id in removed_devices {
    let mut states = device_states.write();
    states.remove(device_id);
}

// 新增的设备：添加到 device_states 并记录连接状态
for device in added_devices {
    let mut states = device_states.write();
    states.insert(
        device.id.clone(),
        crate::DeviceStatus {
            connected: false,
            last_communication: time::Instant::now(),
            error_count: 0,
            reconnect_count: 0,
        },
    );
}
```

#### 更新 worker_map
如果设备列表变化，需要更新 DeviceManager 的 worker_map：

```rust
// 1. 清空 worker_map
self.worker_map.clear();

// 2. 重新填充
for device in &new_config.devices {
    self.worker_map.insert(device.id.clone(), format!("modbus_worker_{}", device.id));
}

// 3. 清空 pending_requests（如果正在处理的设备请求不存在了）
self.pending_requests.clear();
```

### 4. Worker 重启机制

对于受影响的设备，需要重启对应的 ModbusWorker：

**方案一：发送重启请求**
```rust
context.hub().send(Message::DeviceControl {
    device_id: device_id.clone(),
    operation: Operation::Reconnect,
    params: json!({}),
    correlation_id: next_correlation_id(),
});
```

**方案二：优雅重启**
```rust
// ModbusWorker 接收到重启请求后：
// 1. 完成当前操作队列中的任务
// 2. 断开连接
// 3. 等待一段时间后重新连接
```

## 📊 错误处理

### 配置解析错误
```rust
match serde_json::from_str(&config_json) {
    Ok(config) => {
        // 更新配置
        Err(e) => {
            tracing::error!("Failed to parse config: {}", e);
            return;
        }
}
```

### 设备未找到错误
```rust
match self.get_worker_name(&device_id) {
    Some(worker_name) => {
        // 找到，发送消息
    }
    None => {
        tracing::error!("Device {} not found in worker_map", device_id);
        // 返回错误给请求者
    }
}
```

## 🎯 优势

- **无需重启**：配置热更新不需要系统重启
- **平滑更新**：设备状态逐步更新，不会造成中断
- **原子性**：设备列表变化是原子的
- **向后兼容**：可以回滚到旧配置
